# AI Agent Composer Enhancement

Enhanced composer input area for the AI Agent sidebar with attachments, mode toggles, and model selection.

---

## Overview

Replace the current simple input with a rich composer featuring:
- Attachment chips (PDF, DOC, note references)
- Mode toggle buttons (Web search, Thinking mode)
- Model selector dropdown
- File upload button
- Send button

## Component Specifications

### 1. Attachment Chips Area

**Location**: Top of composer, horizontal scrollable row

**Supported Types**:
| Type | Icon | Display | Source |
|------|------|---------|--------|
| PDF | `FileText` (red tint) | `filename.pdf` | File picker |
| DOC/DOCX | `FileText` (blue tint) | `filename.doc` | File picker |
| Note Reference | `StickyNote` | `Note: {title}` | Note picker modal |

**Chip Design**:
```
┌──────────────────────┐
│ 📄 brief.pdf    ×    │
└──────────────────────┘
```
- Background: `bg-stone-100`
- Border: `border border-stone-200`
- Border radius: `rounded-full`
- Padding: `px-3 py-1.5`
- Icon: `h-4 w-4` with type-specific color
- Text: `text-sm text-stone-700 truncate max-w-[120px]`
- Remove button: `×` icon, `text-stone-400 hover:text-stone-600`

**Behavior**:
- Horizontal scroll if many attachments
- Click × to remove
- Hover shows full filename in tooltip
- Empty state: hidden (no attachments row shown)

## State Management

```typescript
interface ComposerState {
  // Input
  message: string

  // Attachments
  attachments: Attachment[]

  // Mode toggles
  webSearchEnabled: boolean
  thinkingModeEnabled: boolean

  // Model
  selectedModel: string

  // Status
  isStreaming: boolean
}

interface Attachment {
  id: string
  type: 'pdf' | 'doc' | 'note'
  name: string
  // For files
  file?: File
  // For note references
  noteId?: string
  noteTitle?: string
}
```

## Interactions

### Adding Attachments

**Files (PDF/DOC)**:
1. Click upload button
2. File picker opens (accept: `.pdf, .doc, .docx`)
3. Selected file added to attachments array
4. Chip appears in attachment row

**Note References**:
1. Type `@` in input OR click dedicated note button (optional)
2. Note picker modal/popover opens
3. Search/select a note
4. Note reference added as attachment chip

### Toggle Behaviors

**Web Search**:
- Click to toggle on/off
- When enabled: icon gets blue background
- State persists during session
- Visual feedback: subtle pulse animation on enable

**Thinking Mode**:
- Click to toggle on/off
- When enabled: icon gets purple background
- State persists during session
- Visual feedback: subtle pulse animation on enable

### Sending Message

1. User types message and/or adds attachments
2. Click send (or press Enter/Cmd+Enter)
3. Composer data packaged:
   ```typescript
   {
     message: string,
     attachments: Attachment[],
     options: {
       webSearch: boolean,
       thinkingMode: boolean,
       model: string
     }
   }
   ```
4. Input clears, attachments clear
5. Mode toggles persist (don't reset)
