Backlinks Section
Purpose
Create a section at the bottom of the note that displays all incoming links — other notes that reference this note. Helps users discover connections and navigate their knowledge graph.

The Prompt
You are a senior frontend developer creating a backlinks section for a PKM app called Memry.

## TASK
Create a backlinks section that displays at the bottom of the note content, showing all notes that link to the current note.

## PROPS INTERFACE
```typescript
interface BacklinksSectionProps {
  backlinks: Backlink[];
  isLoading?: boolean;
  initialCount?: number;        // Default 5, how many to show initially
  collapsible?: boolean;        // Allow collapse entire section
  defaultCollapsed?: boolean;
  sortBy?: 'recent' | 'alphabetical' | 'mentions';
  onSortChange?: (sort: string) => void;
  onBacklinkClick: (noteId: string) => void;
  onShowMore?: () => void;      // For lazy loading
}

interface Backlink {
  id: string;
  noteId: string;
  noteTitle: string;
  noteIcon?: string;            // Emoji or icon
  folder?: string;              // Parent folder name
  date: Date;                   // When link was created or note updated
  mentions: Mention[];          // All mentions of current note
}

interface Mention {
  id: string;
  snippet: string;              // Context around the link
  linkStart: number;            // Position of [[link]] in snippet
  linkEnd: number;
}
```

## OUTPUT

Create a BacklinksSection component that:
1. Displays at bottom of note with clear separation
2. Shows backlink cards with title, date, and context snippet
3. Highlights the link text within snippets
4. Handles multiple mentions per note
5. Shows empty state when no backlinks
6. Supports loading state
7. Implements "show more" for many backlinks
8. Allows collapsing entire section (optional)
9. Supports sorting options (optional)
10. Is fully accessible
11. Navigates to linked note on click

Implementation Notes
Key Techniques Used

Snippet Generation

Backend extracts ~100 characters around each [[link]]
Include enough context to be meaningful
Handle multiple links in same note


Link Highlighting

Parse snippet for link positions
Wrap link text in highlight span
Use dangerouslySetInnerHTML or structured rendering


Virtual Scrolling (Optional)

For notes with 50+ backlinks
Use library like react-virtual
Improves performance significantly


Lazy Loading

Initial load: 5 backlinks
"Show more" loads next batch
Or infinite scroll within section



Why These Choices
ChoiceReasoning48px top marginClear visual break from contentCard-based layoutEach backlink is distinct, clickable unitYellow highlightDraws eye to the actual link, familiar5 initial itemsEnough to be useful, not overwhelmingContext snippetsShows WHY note links, not just that it does
