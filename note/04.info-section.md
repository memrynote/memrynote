Info Section (Collapsible Properties)
Purpose
Create a collapsible properties panel that shows note metadata inherited from the folder template. Users can view, edit, and add properties inline.

The Prompt
You are a senior frontend developer creating a collapsible info/properties section for a PKM app called Memry.

## TASK
Create a collapsible panel that displays note properties with inline editing capabilities.

## INTERACTION BEHAVIORS

### Toggle Expand/Collapse
1. Click header ("▶ Info") → expand panel
2. Click again ("▼ Info") → collapse panel
3. Smooth height animation (200ms)
4. Icon rotates with transition

### Edit Property Value
1. Click on value → enters edit mode
2. Shows appropriate editor (text input, date picker, etc.)
3. Click outside OR press Enter → save and exit
4. Press Escape → cancel, revert to previous
5. Empty value allowed (shows "Empty")

### Add Custom Property
1. Click "+ Add property"
2. Popup opens
3. Enter name, select type
4. Click "Add" → property added to this note only
5. Property appears at end of list

### Delete Custom Property
- Hover on custom property row → show delete icon
- Click delete → remove property
- Only custom (note-specific) properties can be deleted
- Folder-inherited properties cannot be deleted (only emptied)

### Keyboard Navigation
- Tab through property values
- Enter to confirm edit
- Escape to cancel edit
- Arrow up/down in select dropdowns

## PROPS INTERFACE
```typescript
interface InfoSectionProps {
  properties: Property[];               // All properties for this note
  folderProperties: PropertyTemplate[]; // Template from folder
  isExpanded: boolean;                  // Controlled expand state
  onToggleExpand: () => void;
  onPropertyChange: (propertyId: string, value: any) => void;
  onAddProperty: (property: NewProperty) => void;
  onDeleteProperty: (propertyId: string) => void;
  disabled?: boolean;
}

interface Property {
  id: string;
  name: string;
  type: PropertyType;
  value: any;
  icon?: string;
  isCustom: boolean;      // true if added by user, false if from folder
  isRequired?: boolean;
  options?: string[];     // for select/multi-select types
}

type PropertyType =
  | 'text'
  | 'longText'
  | 'number'
  | 'date'
  | 'checkbox'
  | 'select'
  | 'multiSelect'
  | 'rating'
  | 'url'
  | 'relation'
  | 'person';

interface NewProperty {
  name: string;
  type: PropertyType;
}
```

Create an InfoSection component that:
1. Toggles between collapsed and expanded states

3. Supports all property types with appropriate editors
4. Allows inline editing of property values

Implementation Notes
Key Techniques Used

Controlled Expansion

Parent controls isExpanded state
Allows programmatic expand (e.g., when user clicks "Fill properties")


Property Type Registry

Map of property types to editor components
getEditor(type) returns appropriate input component


Animated Height

Use CSS max-height transition or framer-motion
Or grid-template-rows: 0fr → 1fr trick


Inline Editing

Each property value is clickable
Swaps display with input on click
Auto-focus input when entering edit mode
