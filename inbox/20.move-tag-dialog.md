Prompt #20: Move/Tag Dialogs
The Prompt
You are building the Move Dialog and Tag Dialog components for Memry's inbox. These modal dialogs allow users to organize items by moving them to folders and applying tags. They are used from both single-item actions (card hover, preview panel) and bulk actions (Bulk Actions Bar).

## What You Are Building

Two modal dialog components:
1. **Move Dialog** — Select a destination folder for items, with folder creation
2. **Tag Dialog** — Add/remove tags from items, with tag creation

Both dialogs support:
- Single item operations
- Bulk operations (multiple items)
- Search/filter functionality
- Keyboard navigation
- Creating new folders/tags inline

## Move Dialog Props
```typescript
interface MoveDialogProps {
  // State
  isOpen: boolean;
  onClose: () => void;

  // Items being moved
  itemIds: string[];
  itemTitles?: string[];  // For subtitle display

  // Folder data
  folders: FolderTree[];
  currentFolderId?: string;  // Folder items are currently in (to disable)

  // Actions
  onMove: (folderId: string, itemIds: string[]) => Promise<void>;
  onCreateFolder: (name: string, parentId?: string) => Promise<FolderTree>;

  // State
  isMoving: boolean;
}

interface FolderTree {
  id: string;
  name: string;
  itemCount: number;
  children?: FolderTree[];
  isInbox?: boolean;  // Special inbox folder
}
```

## Tag Dialog State Management

### Tracking Changes
```typescript
interface TagDialogState {
  // Original state of tags for each item
  originalItemTags: Map<string, Set<string>>;  // itemId -> tags

  // Current state (after user interactions)
  currentItemTags: Map<string, Set<string>>;

  // Computed: What tags to add/remove when Apply is clicked
  tagsToAdd: Set<string>;      // Tags to add to ALL selected items
  tagsToRemove: Set<string>;   // Tags to remove from ALL selected items
}

function useTagDialogState(
  itemIds: string[],
  initialItemTags: Map<string, Set<string>>
) {
  const [state, setState] = useState<TagDialogState>(() => ({
    originalItemTags: new Map(initialItemTags),
    currentItemTags: new Map(initialItemTags),
    tagsToAdd: new Set(),
    tagsToRemove: new Set(),
  }));

  // Compute tag states for display
  const tagStates = useMemo(() => {
    const allTags = new Set<string>();
    state.currentItemTags.forEach(tags => {
      tags.forEach(tag => allTags.add(tag));
    });

    const result = new Map<string, 'all' | 'some' | 'none'>();

    allTags.forEach(tag => {
      let hasCount = 0;
      state.currentItemTags.forEach(tags => {
        if (tags.has(tag)) hasCount++;
      });

      if (hasCount === 0) {
        result.set(tag, 'none');
      } else if (hasCount === itemIds.length) {
        result.set(tag, 'all');
      } else {
        result.set(tag, 'some');
      }
    });

    return result;
  }, [state.currentItemTags, itemIds.length]);

  // Toggle a tag
  const toggleTag = useCallback((tag: string) => {
    setState(prev => {
      const currentState = tagStates.get(tag) || 'none';
      const newItemTags = new Map(prev.currentItemTags);

      if (currentState === 'all') {
        // Remove from all
        newItemTags.forEach((tags, itemId) => {
          const newTags = new Set(tags);
          newTags.delete(tag);
          newItemTags.set(itemId, newTags);
        });
      } else {
        // Add to all (covers 'none' and 'some')
        newItemTags.forEach((tags, itemId) => {
          const newTags = new Set(tags);
          newTags.add(tag);
          newItemTags.set(itemId, newTags);
        });
      }

      // Compute changes from original
      const { tagsToAdd, tagsToRemove } = computeChanges(
        prev.originalItemTags,
        newItemTags
      );

      return {
        ...prev,
        currentItemTags: newItemTags,
        tagsToAdd,
        tagsToRemove,
      };
    });
  }, [tagStates]);

  // Create and add a new tag
  const createAndAddTag = useCallback((tagName: string) => {
    const normalizedTag = normalizeTagName(tagName);

    setState(prev => {
      const newItemTags = new Map(prev.currentItemTags);

      // Add to all items
      newItemTags.forEach((tags, itemId) => {
        const newTags = new Set(tags);
        newTags.add(normalizedTag);
        newItemTags.set(itemId, newTags);
      });

      const { tagsToAdd, tagsToRemove } = computeChanges(
        prev.originalItemTags,
        newItemTags
      );

      return {
        ...prev,
        currentItemTags: newItemTags,
        tagsToAdd,
        tagsToRemove,
      };
    });
  }, []);

  const hasChanges = state.tagsToAdd.size > 0 || state.tagsToRemove.size > 0;

  return {
    tagStates,
    tagsToAdd: state.tagsToAdd,
    tagsToRemove: state.tagsToRemove,
    hasChanges,
    toggleTag,
    createAndAddTag,
  };
}

function normalizeTagName(input: string): string {
  return input
    .toLowerCase()
    .trim()
    .replace(/^#/, '')           // Remove leading #
    .replace(/\s+/g, '-')        // Replace spaces with hyphens
    .replace(/[^a-z0-9\-_]/g, '') // Remove invalid chars
    .slice(0, 30);               // Max length
}
```

## Keyboard Navigation

### Both Dialogs
```typescript
const DIALOG_KEYBOARD_SHORTCUTS = {
  'Escape': 'Close dialog (discard changes)',
  'Enter': 'Apply/Confirm (when no input focused)',
  'Tab': 'Move focus to next element',
  'Shift+Tab': 'Move focus to previous element',
  'ArrowUp': 'Previous item in list',
  'ArrowDown': 'Next item in list',
  'Space': 'Toggle selected item (folder/tag)',
};

function useDialogKeyboard(
  isOpen: boolean,
  onClose: () => void,
  onConfirm: () => void,
  listRef: RefObject<HTMLDivElement>
) {
  const [focusedIndex, setFocusedIndex] = useState(-1);

  useEffect(() => {
    if (!isOpen) {
      setFocusedIndex(-1);
      return;
    }

    const handleKeyDown = (e: KeyboardEvent) => {
      switch (e.key) {
        case 'Escape':
          e.preventDefault();
          onClose();
          break;

        case 'Enter':
          // Only confirm if not in an input
          if (!(e.target instanceof HTMLInputElement)) {
            e.preventDefault();
            onConfirm();
          }
          break;

        case 'ArrowDown':
          e.preventDefault();
          setFocusedIndex(prev => prev + 1);
          break;

        case 'ArrowUp':
          e.preventDefault();
          setFocusedIndex(prev => Math.max(-1, prev - 1));
          break;

        case ' ':
          // Space to toggle (only if not in input)
          if (!(e.target instanceof HTMLInputElement) && focusedIndex >= 0) {
            e.preventDefault();
            // Trigger selection of focused item
          }
          break;
      }
    };

    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
  }, [isOpen, onClose, onConfirm, focusedIndex]);

  return { focusedIndex, setFocusedIndex };
}
```

### Focus Management

4. When dialog closes: Return focus to trigger element

```typescript
function useFocusTrap(dialogRef: RefObject<HTMLDivElement>, isOpen: boolean) {
  useEffect(() => {
    if (!isOpen || !dialogRef.current) return;

    const focusableElements = dialogRef.current.querySelectorAll(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );

    const firstElement = focusableElements[0] as HTMLElement;
    const lastElement = focusableElements[focusableElements.length - 1] as HTMLElement;

    const handleTab = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return;

      if (e.shiftKey) {
        if (document.activeElement === firstElement) {
          e.preventDefault();
          lastElement.focus();
        }
      } else {
        if (document.activeElement === lastElement) {
          e.preventDefault();
          firstElement.focus();
        }
      }
    };

    document.addEventListener('keydown', handleTab);

    // Focus first element on open
    firstElement?.focus();

    return () => document.removeEventListener('keydown', handleTab);
  }, [isOpen, dialogRef]);
}
```

## Output

Create two React components:

1. **MoveDialog** — Modal for moving items to folders
2. **TagDialog** — Modal for managing item tags

Use Tailwind CSS for styling. Both components should:
1. Render as modal dialogs with backdrop
2. Include search/filter functionality
3. Support keyboard navigation
4. Allow creating new folders/tags inline
5. Show appropriate loading/empty states
6. Handle focus management correctly
7. Work responsively (bottom sheet on mobile)

Also create supporting hooks:
- useFolderSelection for Move dialog state
- useTagDialogState for Tag dialog state (with change tracking)
- useFocusTrap for accessibility
- useDialogKeyboard for keyboard navigation

Implementation Notes
Key Techniques Used:
TechniqueWhyChange tracking for tagsUsers see pending changes before applyingNested folder displayVisual hierarchy helps navigationInline creationUsers don't leave context to create new folder/tagSearch with createType to filter, create if not foundFocus trapRequired for modal accessibility
Design Choices:

Separate "Selected Tags" section — Makes it clear which tags are currently applied. User immediately sees current state.
Three-state checkboxes for tags — "All have", "some have", "none have" covers all bulk tagging scenarios without confusion.
Create inline, not separate flow — User can create a folder/tag without leaving the dialog. Smoother workflow.
Change summary in footer — User knows what will happen when they click Apply. Reduces anxiety.
Search doubles as create — Same input for finding existing and creating new. Common pattern (Notion, Linear).
